## สไลด์ 1 คำถาม

# multi-agent คุ้มกว่า single agent ไหม?

**กรณีศึกษา:** ตรวจจับประชด/เสียดสีภาษาไทย

วัด 4 มิติพร้อมกัน ไม่ใช่แค่ F1
> **คุณภาพ · ค่าใช้จ่าย · เวลา · จำนวน LLM calls**

*(พูด: คนมักโชว์แต่ F1 แล้วบอกว่า multi-agent ดีกว่า แต่ไม่เคยบอกว่าจ่ายเท่าไหร่
และไม่เคยเทียบกับ single agent ที่ถูกรีดจนสุด เดี๋ยวจะเห็นว่านั่นคือจุดพลิก)*

## สไลด์ 2 ทำไมต้องประชด

ประชด = **เจตนาจริงตรงข้ามกับความหมายผิวเผิน**

**กฎสำคัญของงานนี้: ต้องมี "การเสแสร้ง"**
- ✅ "ขอบคุณมากนะคะที่ให้รอ 2 ชั่วโมง" → ประชด (แกล้งขอบคุณ)
- ❌ "ร้านนี้ห่วยมาก รอนาน" → **บ่นตรงๆ = ไม่ประชด**
- ❌ "อาหารอร่อย แต่ที่จอดรถแย่" → **รีวิวสมดุล = ไม่ประชด**

*(สองข้อล่างคือหัวใจ เดี๋ยวจะเห็นว่า LLM พังตรงนี้)*

## สไลด์ 3 ชุดข้อมูล + ข้อจำกัดที่ผมประกาศเอง

**gold = 127 ข้อ** (ประชด 30 / ไม่ประชด 97)
Wongnai (รีวิวยาว) + Wisesight (ทวีตสั้น) · คนตัดสินขั้นสุดท้ายแบบ blind

### ⚠️ Self-selection bias ผมพูดก่อนโดนถาม
ประชด 30 ข้อ **ถูก GPT-4o ช่วยขุดมาก่อน** (คัด 800 → 495)
→ **recall ของ GPT-4o ในงานนี้สูงเกินจริง**
→ **แต่ bias กระทบทุกระบบเท่ากัน → การ *เทียบ* ยังยุติธรรม**

**หลักฐาน:** ข้อที่ LLM ขุดมาเอง มี FPR = **1.000 (9/9)** vs ข้อจากคีย์เวิร์ด 0.205

### ⚠️ n(ประชด) = 30 → CI กว้าง (±0.10) จำเลขนี้ไว้ มันคือพระเอกของเรื่อง

## สไลด์ 6 Finding 1: สมมติฐานผม **ผิด**

### ผมคิดว่า LLM จะ "หาประชดไม่เจอ"

**ผลจริง: recall = 1.000 จับครบทุกข้อ ไม่พลาดเลยสักข้อ**

|  | ทายประชด | ทายไม่ประชด |
|---|---|---|
| ประชดจริง | TP **30** | FN **0** |
| ไม่ประชดจริง | FP **27** | TN 70 |

### คอขวดจริงคือ **precision = 0.526**
FP 27 ข้อส่วนใหญ่ = **"รีวิวสมดุล"** (ชมจริง + ติจริง)
→ LLM แยกไม่ออกระหว่าง **"มีทั้งบวกและลบ"** กับ **"แกล้งชมเพื่อเหน็บ"**

## สไลด์ 7 Finding 2 ⭐ **การออกแบบ verifier สำคัญกว่าการมี verifier**

**verifier ตัวเดียวกัน สถาปัตยกรรมเดียวกัน ต่างแค่กฎตอนก้ำกึ่ง**

| | v1 "ไม่ชัด → **ทิ้ง**" | v2 "ไม่ชัด → **เก็บ**" |
|---|---|---|
| พลิกทิ้ง | 27 ข้อ | 8 ข้อ |
| ฆ่า FP ถูก | 17 | 7 |
| **ฆ่า TP ผิด** | **10** ❌ | **1** ✅ |
| recall | 0.667 | **0.967** |
| F1 | 0.714 | **0.744** |

**เพราะ ประชดที่แนบเนียน อ่านได้สองแง่โดยธรรมชาติ → "ไม่ชัดให้ทิ้ง" ขัดกับงานนี้ตรงๆ**

## สไลด์ 8 จุดพลิก 🔴 baseline ถูกมัดมือมาตลอด

### GPT ไม่ได้ตอบ "0/1" มันตอบ **token ที่มี logprob**

เราทิ้ง logprob ทุกข้อ แล้วเอา 0/1 ดิบไปเทียบ = **ให้ multi-agent สู้กับคู่ต่อสู้ที่ถูกมัดมือ**

**อ่าน P("ประชด") จาก logprob แล้วปรับ threshold** (leave-fold-out, ไม่มี leak):

| | F1 | cost | calls เพิ่ม |
|---|---|---|---|
| baseline @argmax (เดิม) | 0.690 | $0.094 |, |
| **baseline + threshold** | **0.725** | **$0.094** | **0 (ฟรี!)** |

**+0.035 F1 ฟรี ไม่เพิ่ม call แม้แต่ครั้งเดียว** (คะแนนได้มาจาก call ที่จ่ายไปแล้ว)

## สไลด์ 9 Finding 3 ⭐⭐ เทียบใหม่แบบยุติธรรม → **ไม่มีใครชนะ**

**เทียบทุกระบบกับ baseline+threshold (0.725) แทน baseline ดิบ · paired bootstrap 5,000 รอบ**

| ระบบ | ΔF1 | 95% CI | P(ไม่ดีกว่า) | cost |
|---|---|---|---|---|
| v2 (multi-agent ดีที่สุด) | +0.019 | **[−0.073, +0.113]** | 36% | 1.80× |
| hybrid | −0.025 | [−0.117, +0.071] | 70% | 4.3× |
| debate | −0.030 | [−0.158, +0.096] | 69% | 7.4× |

> **ทุกระบบ CI คร่อม 0 ไม่มี multi-agent ตัวไหนแยกจากเอเจนต์เดี่ยวที่ tune แล้วได้**
> ข้อได้เปรียบ v2 เดิม "+0.054 vs baseline" → เหลือ **+0.019** → **ครึ่งหนึ่งคือ baseline ถูก handicap**

## สไลด์ 10 Finding 4 ⭐⭐⭐ แกนที่ขยับ cost จริงคือ **โมเดล** ไม่ใช่สถาปัตยกรรม

**เอเจนต์เดี่ยว + threshold บน 5 โมเดล (ราคาต่างกัน 25×):**

| โมเดล | F1 | $/รัน |
|---|---|---|

## สไลด์ 11 สรุป (แก้ใหม่หลังเทียบยุติธรรม)

### เรื่องเก่าที่ผมเกือบเชื่อ:
~~"multi-agent ดีกว่า +0.054 คุ้มถ้า FP แพง"~~

### เรื่องจริงหลังรีด baseline จนสุด:
1. ⭐ **ก่อนถามว่า "เพิ่ม agent ไหม" → ถามก่อนว่า "รีดของที่ agent เดียวให้มาแล้วหมดยัง"**
   baseline ทิ้ง logprob ที่จ่ายเงินซื้อมาแล้วทุกข้อ
2. **บนงานนี้ ไม่มีหลักฐานว่า multi-agent ใดคุ้มค่าเงินที่จ่ายเพิ่ม** (ทุก CI คร่อม 0)
3. **"cost-aware" ตัวจริง = เลือกโมเดลถูก + อ่าน logprob** ไม่ใช่เลือกสถาปัตยกรรม

### ข้อควรระวัง (ไม่ใช่ว่า multi-agent ไร้ค่า):
error profile ต่างกัน v2 recall 0.967 / threshold 0.833 → งาน **"ห้ามพลาด"** v2 ยังคุ้ม
"แยกกันไม่ออก" ≠ "เท่ากันแน่" **n=30 คือเพดาน**

## สไลด์ 12 งานต่อ

### เรียงตามผลตอบแทน
1. ⭐ **ขยาย gold เป็น 50-60 ประชด** ← ทางเดียวที่ทำให้ทุก CI แคบลงพอจะฟันธง
   ใช้ logprob คัด "ข้อก้ำกึ่ง" (P 0.2-0.8) มา label ได้ข้อที่แยกระบบได้จริง ไม่เสียแรงกับข้อชัดๆ
2. **แก้ precision ที่ต้นทาง** ใส่ตัวอย่าง "รีวิวสมดุล = 0" ใน detector prompt (ฟรี ไม่เพิ่ม call)
3. **cascade (WCB คัดกรอง) ลองแล้ว แพ้** (finding 6): WCB "จัดอันดับไม่เป็น" คัดกรองไม่ได้

## คำถามที่น่าจะโดนถาม (เตรียมไว้)

**Q: ในเมื่อ multi-agent ไม่ชนะ แล้วทำมาทำไม?**
> เพราะ "ทำแล้วพบว่าไม่คุ้ม" คือผลลัพธ์ที่มีค่า และผมได้บทเรียนที่แข็งกว่าเดิม:
> ต้อง tune baseline ให้สุดก่อน ไม่งั้นเทียบเอนเอียง งานส่วนใหญ่ในสายนี้ข้ามขั้นนี้ไป

**Q: gold น้อยไป 127 ข้อ เชื่อได้เหรอ?**
> ครับ ผมถึงแนบ CI ทุกตัวเลข และผมบอกเองว่า n=30 คือเพดาน ทุกข้อสรุปของผมคือ
> "แยกกันไม่ออก" ไม่ใช่ "เท่ากันแน่นอน" งานต่อลำดับแรกคือขยาย gold

**Q: ใช้ GPT-4o ขุดข้อมูลแล้วเอามาวัด ไม่ cheat เหรอ?**
> cheat สำหรับ **ตัวเลขดิบ** (โดยเฉพาะ recall) แต่ไม่ cheat สำหรับ **การเทียบระบบ**
> เพราะทุกระบบกิน bias ก้อนเดียวกัน · หลักฐาน: กองที่ LLM ขุดเอง FPR = 1.000

**Q: threshold tuning มันคือ overfit ไหม?**
> ไม่ เลือก threshold แบบ leave-fold-out (fold k ใช้ค่าจากอีก 4 folds) ทุกตัวเลขที่รายงาน
> ไม่มีข้อไหนมีส่วนกำหนด threshold ที่ตัดสินตัวมันเอง

**Q: ราคาโมเดลเชื่อได้แค่ไหน?**
> token counts วัดจริงจาก API เป๊ะ · ราคา/1M เป็นค่าที่ต้องเช็คกับราคาปัจจุบัน (อยู่ใน frontier.py)
