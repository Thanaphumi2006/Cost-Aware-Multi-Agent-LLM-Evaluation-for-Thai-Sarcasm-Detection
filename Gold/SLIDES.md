## สไลด์ 2 — ทำไมต้องประชด

ประชด = **เจตนาจริงตรงข้ามกับความหมายผิวเผิน**

**กฎสำคัญของงานนี้: ต้องมี "การเสแสร้ง"**
- ✅ "ขอบคุณมากนะคะที่ให้รอ 2 ชั่วโมง" → ประชด (แกล้งขอบคุณ)
- ❌ "ร้านนี้ห่วยมาก รอนาน" → **บ่นตรงๆ = ไม่ประชด**
- ❌ "อาหารอร่อย แต่ที่จอดรถแย่" → **รีวิวสมดุล = ไม่ประชด**

*(สองข้อล่างคือหัวใจ — เดี๋ยวจะเห็นว่า LLM พังตรงนี้)*

## สไลด์ 3 — ชุดข้อมูล + ข้อจำกัดที่ผมประกาศเอง

**gold = 127 ข้อ** (ประชด 30 / ไม่ประชด 97)
Wongnai (รีวิวยาว) + Wisesight (ทวีตสั้น) · คนตัดสินขั้นสุดท้ายแบบ blind

### ⚠️ Self-selection bias — ผมพูดก่อนโดนถาม
ประชด 30 ข้อ **ถูก GPT-4o ช่วยขุดมาก่อน** (คัด 800 → 495)
→ **recall ของ GPT-4o ในงานนี้สูงเกินจริง**
→ **แต่ bias กระทบทุกระบบเท่ากัน → การ *เทียบ* ยังยุติธรรม**

**หลักฐาน:** ข้อที่ LLM ขุดมาเอง มี FPR = **1.000 (9/9)** vs ข้อจากคีย์เวิร์ด 0.205

## สไลด์ 6 — Finding 1: สมมติฐานผม **ผิด**

### ผมคิดว่า LLM จะ "หาประชดไม่เจอ"

**ผลจริง: recall = 1.000 — จับครบทุกข้อ ไม่พลาดเลยสักข้อ**

|  | ทายประชด | ทายไม่ประชด |
|---|---|---|
| ประชดจริง | TP **30** | FN **0** |
| ไม่ประชดจริง | FP **27** | TN 70 |

### คอขวดจริงคือ **precision = 0.526**
FP 27 ข้อส่วนใหญ่ = **"รีวิวสมดุล"** (ชมจริง + ติจริง)

→ LLM แยกไม่ออกระหว่าง **"มีทั้งบวกและลบ"** กับ **"แกล้งชมเพื่อเหน็บ"**

## สไลด์ 7 — Finding 2 ⭐ **การออกแบบ verifier สำคัญกว่าการมี verifier**

**verifier ตัวเดียวกัน สถาปัตยกรรมเดียวกัน — ต่างแค่กฎตอนก้ำกึ่ง**

| | v1 "ไม่ชัด → **ทิ้ง**" | v2 "ไม่ชัด → **เก็บ**" |
|---|---|---|
| พลิกทิ้ง | 27 ข้อ | 8 ข้อ |
| ฆ่า FP ถูก | 17 | 7 |
| **ฆ่า TP ผิด** | **10** ❌ | **1** ✅ |
| recall | 0.667 | **0.967** |
| F1 | 0.714 | **0.744** |

**v1 เผา TP ทิ้งเกือบเท่าที่มันฆ่า FP → F1 แทบไม่ขยับ ทั้งที่จ่ายค่าโทเคนเท่ากัน**

### เพราะอะไร?
**ประชดที่แนบเนียน อ่านได้สองแง่โดยธรรมชาติ**
"ถ้าไม่ชัดให้ทิ้ง" จึงเป็นกฎที่**ขัดกับธรรมชาติของงานนี้โดยตรง**

## สไลด์ 8 — มันดีกว่าจริงไหม? (สถิติ)

**ทั้งสองระบบวัด gold ชุดเดียวกันข้อต่อข้อ → ต้องใช้ paired test**
*(ไม่ใช่เทียบว่า F1 เกินขอบบน CI ของ baseline — นั่นเข้มเกินไปผิดๆ)*

### paired bootstrap 5,000 รอบ
- ผลต่าง F1 = **+0.054**
- 95% CI = **[−0.002, +0.113]**
- โอกาสที่ v2 ไม่ดีกว่า = **3%**
- **McNemar: v2 ถูก/base ผิด = 8 | base ถูก/v2 ผิด = 1**

> **มั่นใจ ~97% ว่าดีกว่าจริง — แต่พลาดเกณฑ์ 95% เป๊ะไป 0.002**
> **สาเหตุ: ประชดมีแค่ n=30 → กำลังสถิติไม่พอ**
> **นี่คือเพดานทางสถิติ ไม่ใช่เพดานทางเทคนิค**

## สไลด์ 9 — สรุป + งานต่อ

### สรุป
1. **multi-agent ดีกว่าจริง แต่ marginal** — F1 +0.054 แลกกับ **cost 1.80×**
   → คุ้มเฉพาะเมื่อ **false positive มีราคาแพง**
2. ⭐ **การมี verifier ไม่การันตีว่าดีขึ้น** — prompt ของแต่ละด่านสำคัญกว่าสถาปัตยกรรม
3. **อย่ารายงาน accuracy** — baseline acc 0.787 vs เดา 0 ทุกข้อ = 0.764 (**ต่างกัน 2 จุด!**)

### งานต่อ (เรียงตามผลตอบแทน)
1. **ขยาย gold เป็น 50–60 ประชด** ← ทางเดียวที่ทำให้ฟันธงได้ (เหลือ ~470 ข้อยังไม่ตรวจ)
2. **WangchanBERTa** — โมเดลเล็กพอไหม / ต้องใช้ LLM จริงหรือ
3. **แก้ precision ที่ต้นทาง** — ใส่ตัวอย่าง "รีวิวสมดุล = 0" ใน detector prompt
   *(ถูกกว่าเพิ่ม verifier — ไม่เพิ่ม call เลย)*

## คำถามที่น่าจะโดนถาม (เตรียมไว้)

**Q: gold น้อยไป 127 ข้อ เชื่อได้เหรอ?**
> ครับ ผมถึงแนบ CI ทุกตัวเลข ไม่ใช่ point estimate เดี่ยว และผมบอกเองว่า n=30 คือเพดาน
> ที่ทำให้ฟันธงไม่ได้ — งานต่อลำดับแรกของผมคือขยาย gold ไม่ใช่จูน prompt เพิ่ม

**Q: ใช้ GPT-4o ขุดข้อมูลแล้วเอา GPT-4o มาวัด ไม่ cheat เหรอ?**
> cheat ครับ — สำหรับ **ตัวเลขดิบ** โดยเฉพาะ recall
> แต่ไม่ cheat สำหรับ **การเทียบระบบ** เพราะทุกระบบกิน bias ก้อนเดียวกัน
> และผมมีหลักฐานว่ามันโผล่ที่ไหน: กองที่ LLM ขุดเอง FPR = 1.000

**Q: ทำไมไม่ใช้ accuracy?**
> เพราะข้อมูลเอียง 76/24 — เดาว่า "ไม่ประชด" ทุกข้อได้ 0.764 ฟรีๆ
> accuracy จะซ่อน FP 27 ข้อไว้ทั้งหมด

**Q: ทำไม verifier ไม่ยิงทุกข้อ?**
> verifier พลิกได้ทางเดียว (1→0) ยิงกับข้อที่ detector ตอบ 0 อยู่แล้วก็เปลืองโทเคนเปล่า
> → calls = 183 แทนที่จะเป็น 254 (ประหยัด 28%)
