สคริปต์รอบที่ 1 "ระบุ / ให้คะแนน" ข้อความยังไง
================================================================
สรุป: ไม่มีโมเดล ไม่มี AI — เป็นแค่การ "นับสัญญาณด้วยกฎ" ล้วนๆ


■ หัวใจ: อ่านทีละข้อความ แล้วบวกคะแนนเมื่อเจอสัญญาณ
----------------------------------------------------------------
สคริปต์มีลิสต์คำ/แพตเทิร์นที่กำหนดไว้ แล้วเช็กว่าข้อความมีสัญญาณพวกนี้ไหม
เจออันไหนก็บวกแต้ม:

  สัญญาณ            | เช็กยังไง                                  | ได้กี่แต้ม
  ------------------|--------------------------------------------|----------
  คำชม              | มีคำใน [ดี, เยี่ยม, สุดยอด, เลิศ, ฟิน...]   |   +1
  อิโมจิเสียดสี     | มี 🙄 🙃 👏 ✨ ไหม                          |   +2
  หัวเราะ 555       | มีเลข 5 ติดกัน 3 ตัวขึ้นไป                  |   +1
  ยืดตัวอักษร       | มีตัวอักษรซ้ำ 3 ตัวติด (เช่น มาก "กก")      |   +1
  ขอบคุณ            | มีคำว่า ขอบคุณ/ขอบใจ                        |   +1
  ชม + บริบทแย่     | มีคำชม "และ" คำลบ (รอ/นาน/พัง/แย่) พร้อมกัน |   +3  <- แรงสุด

  คะแนนรวม = ผลบวกของทุกสัญญาณที่เจอในข้อความนั้น


■ ตัวอย่างการนับจริง
----------------------------------------------------------------
ข้อความ: "ร้านนี้ดีย์มากกก สั่งไป 40 นาทีของยังไม่มา 👏"
  - เจอคำชม "ดี"                        -> +1
  - เจออิโมจิ 👏                        -> +2
  - เจอตัวอักษรซ้ำ "กกก"                -> +1
  - เจอคำชม "ดี" + คำลบ "ไม่มา" พร้อมกัน -> +3
  - รวม = 7  -> เกินเกณฑ์ (3) -> เข้ากอง "สงสัยสูง"

ข้อความ: "อร่อยจริง ราคาไม่แพง จะกลับมาอีก"
  - ไม่มีคำในลิสต์เลย -> 0 แต้ม -> กองปกติ


■ ทำไมส่วนใหญ่ได้ 0-1
----------------------------------------------------------------
เพราะข้อความทั่วไปไม่มีสัญญาณพวกนี้ (ไม่มีคำชมเว่อร์ ไม่มีอิโมจิ ไม่มี 555)
มันเลยได้ 0 หรือมีแค่สัญญาณเดียวได้ 1 -> คนหมู่มากอยู่ตรงนี้
ส่วนพวกที่มีหลายสัญญาณพร้อมกันถึงจะเด้งขึ้นไปคะแนนสูง และถูกดึงมาเป็นผู้ต้องสงสัย


■ "ระบุ" ในที่นี้ = คัดผู้ต้องสงสัย ไม่ใช่ตัดสินว่าประชด
----------------------------------------------------------------
ย้ำจุดสำคัญสุด: คะแนนสูง "ไม่ได้แปลว่าประชด"
มันแปลว่า "มีสัญญาณเยอะพอที่คนควรเอาไปดู" เท่านั้น
ตัวตัดสินจริงว่าประชดหรือไม่ ยังต้องเป็นคนติดป้าย

เปรียบเทียบ: เหมือนเครื่องตรวจโลหะที่ดังเมื่อเจอโลหะ
แต่ไม่ได้บอกว่าเป็นทองหรือเหรียญ — คนต้องมาเปิดดูเอง


■ ผลรันล่าสุด (ตีความ)
----------------------------------------------------------------
- ข้อความส่วนใหญ่ได้ 0-1 (ไม่มีสัญญาณ) = ปกติ
- กองคะแนนสูงคือผู้ต้องสงสัยที่ถูกดึงมา
- คัดได้ 400 ข้อ ผสมสงสัยสูง 240 / ปกติ 160 ตามที่ตั้งไว้ = ทำงานตรงสเปก

ข้อควรระวังที่เห็นจากผล: รีวิว Wongnai เป็นข้อความยาว จึงมักมีทั้งคำชมและคำลบ
ในเรื่องเดียวโดยธรรมชาติ (เล่าว่ารอนานแต่อาหารดี) ทำให้ถูกจับเป็น "ชม+แย่"
ทั้งที่เป็นรีวิวปกติ ไม่ได้ประชด -> เป็นเรื่องปกติของการคัดด้วยคีย์เวิร์ด
ตัวจริงให้คนติดป้าย 0 ไป หรือปรับสคริปต์ (ตัดข้อความยาว / เน้น Wisesight ที่สั้นกว่า)
